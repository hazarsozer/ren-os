§ 02 · Memory architecture

Four layers,
each scoped tight.

Off-the-shelf tools waste tokens. The framework's memory model layers four mechanisms with deliberately different jobs — and treats the prompt cache as sacred (Anthropic monitors cache hit rate at SEV level; we honor that).

iii. · iv. for scale
Within session Context Mode

Sandboxes tool output. A 315 KB Playwright snapshot becomes a 5.4 KB compact summary the model actually reads.

98 %output reduction
Cross session claude-mem

Compressed observations across sessions. SQLite + ChromaDB + ONNX embeddings, three-layer progressive disclosure retrieval.

autoper session-end
Personal synthesis Your wiki

Hand-curated markdown. Decisions, patterns, lessons — the friend's own knowledge graph. Hierarchical: one master + a sub-wiki per project.

signalonly · zero default edits
Wiki at scale qmd

Hybrid BM25 + vector + LLM re-rank for when the wiki crosses 200 pages. Drops in cleanly because v1 already enforces plain markdown + stable frontmatter.

v2upgrade path
Cross friend Activity Feed

The only layer that crosses friend boundaries. Deliberately terse — couple sentences per session-end, only project + task + files. Format itself is the privacy mechanism.

opt-inper session
A constraint we honor

Never the system prompt.

The wake-up hook injects wiki context into the conversation layer, never the system prompt. Cached tokens cost 10 % of normal input; modifying the system prompt mid-session breaks the prefix and the whole cache evaporates.

A budget we live in

3 – 5 K tokens at wake-up.

Master index + project index + last 10 log entries + session pointer + recent friend activity. Never the whole wiki. The remaining budget belongs to the friend's actual work.

§ 03 · The daily loop

Open the laptop.
Close the laptop.

The framework's value shows up at session boundaries. Start of session: wake-up loads what matters. During: hooks run silently while you work. End: /sf:wrap promotes high-signal moments to the wiki and writes a terse line to the Activity Feed.

Three phases.

What the wake-up hook does

  1. Reads pwd to determine which project you're in.
  2. Loads master wiki/index.md (small catalog).
  3. If you're in a project: loads its index.md, CONTEXT.md, and the last 10 log.md entries.
  4. Loads the master log.md tail.
  5. Pulls the Activity Feed; skims other friends' recent log entries.
  6. Composes one context-injection message in the conversation layer.
  7. Posts a new entry in your <handle>.log.md: start · working in ...
  8. Pushes the Activity Feed update to GitHub.
What you actually see

Before your first prompt, you see a single message: "Framework wake-up context." It's mostly index entries, last 10 log lines, the session pointer from your last wrap, and a couple lines about what your friends touched recently.

You were working on JWT auth middleware. Open thread: write tests for the expired-token edge case.
# Recent friend activity › friend-b · yesterday · landing page copy on Sidecar › friend-c · 3 days ago · Restore onboarding debug

What runs while you work

  1. Context Mode sandboxes every tool output — keeps the conversation lean.
  2. claude-mem quietly captures observations for next session's recall.
  3. Superpowers skills activate as work matches them: brainstorming, TDD, debugging, code-review.
  4. context7 fetches up-to-date version-specific docs for libraries you touch.
  5. claude-md-management audits CLAUDE.md if you edit it.
  6. /sf:note "..." — pin a thought for /sf:wrap to consider later.
  7. /sf:recall "..." — query the wiki mid-session without loading more.
Designed not to interrupt

The framework's daily-loop discipline: don't ask the friend to do anything new. Hooks coordinate themselves. Skills activate themselves. The friend builds the thing they sat down to build.

"Mature harness work looks less like building structure up and more like pruning it down." — PY, harness research
$ /sf:note "JWT secret rotates weekly" ✓ pinned for /sf:wrap $ /sf:recall "refresh-token approach" → wiki/decisions/auth-refresh-flow.md

What /sf:wrap does

  1. Reads the session log (CC's record + any /sf:note pins).
  2. Applies a high-signal threshold: real decision, pattern, or lesson? If not, zero edits. That's the discipline.
  3. If signal exists: updates affected pages — diffs shown for approval.
  4. Updates CONTEXT.md with the session pointer for next time.
  5. Writes the terse Activity Feed entry — never code, never reasoning.
  6. Commits and pushes the feed.
  7. Returns a brief summary.
The terse format itself is the firewall

Project + task + files. That's the entire shape of a session-end entry. The format constraint prevents accidental leakage by default — you don't need a diff-review safety net.

For sensitive work: /sf:wrap --skip-feed or SF_SKIP_FEED=1.

## [18:45] end · hazar · session complete Worked on Dev/sidecar/ — wrote JWT expired-token tests and fixed off-by-one in token validation. Touched: tests/test_auth.py, api/auth.py.

§ 05 · Curated stack

Eight plugins.
Every one earns its slot.

Six required at install, one conditional, one documented-not-bundled. The framework's value is what it leaves out. Each plugin is judged on two axes: tool fit, and provider trust (the meme-coin rug-pull incident taught us popularity ≠ safety).

6 + 1 + 1
Required at install six plugins · methodology, memory, efficiency, docs

Superpowers

obra

The development methodology layer. Seven-phase workflow — brainstorm → worktree → plan → TDD → subagent → review → finish. Powered the brainstorming session this design came out of.

MIT · official marketplace

Skill Creator

Anthropic

Skill authoring + Layer 1 description optimizer. 60 / 40 train-test split, 3 runs per query, 5 iterations. Foundation of the framework's two-layer self-improvement loop.

Apache 2.0

claude-mem

thedotmack

Cross-session memory. SQLite + ChromaDB + three-layer progressive-disclosure retrieval. Two independent practitioner recommendations · most-cited tool for the job.

Apache 2.0

Context Mode

mksglu

Within-session token efficiency. Sandboxes tool output via subprocess isolation. 315 KB → 5.4 KB (98 % reduction). No daemon.

ELv2 · personal / team use

context7

Upstash

Version-aware documentation lookup. Solves "Claude wrote code against an outdated version of the library." Auto-activating; cheap sub-agent on Sonnet for docs research.

Permissive · Upstash key needed

claude-md-management

Anthropic

CLAUDE.md hygiene + /revise-claude-md. Captures session learnings into project CLAUDE.md. Complements /sf:wrap at a different layer (CLAUDE.md vs. wiki).

Permissive · 205K installs
Conditional · asked at onboarding one plugin · UI projects only

Frontend Design

Anthropic

Auto-activating skill for distinctive frontend work. Rejects generic AI aesthetics; teaches bold typography and color choices. The friend gets asked at /sf:install — skipped for backend-only groups.

Permissive · skipped if no UI
Documented · not bundled one plugin · install on demand

Ralph

Anthropic

The canonical lightweight autonomous-loop pattern. Stop-hook session loop for overnight runs. Not bundled — its Stop hook would compete with claude-mem's lifecycle hooks. Documented; install on demand.

Permissive · install on demand
Explicitly rejected — and why
ECC (Everything Claude Code) GSD Redux MCP Agent Mail Mercury / Proton SaaS Mem0 / Letta / Zep / LangMem lean-ctx framework-level secret scanning

01

Environment check

Before anything else: confirm the friend's machine has the prerequisites the framework assumes.

  • Claude Code 1.0.33+, Node 22.5+, git, gh CLI
  • claude auth login + gh auth status both healthy
  • ANTHROPIC_API_KEY in environment
  • Upstash API key (for context7 docs lookups)
  • OS: Linux or macOS primary; Windows via WSL
02

Required plugins

Six required plugins installed in hook-coordination order — most specific first, most general last.

  • Context Mode — sandboxes tool output first
  • claude-mem — cross-session memory second
  • Superpowers — methodology skills
  • Skill Creator — authoring + L1 optimizer
  • context7 — version-aware docs
  • claude-md-management — CLAUDE.md hygiene
03

Activity Feed + conditional plugins

Cross-friend visibility wired up; conditional plugins offered.

  • Activity Feed: shared private GitHub repo URL → clone → identities setup
  • Ask: Frontend Design? (UI work)
  • Ask: Ralph documented for overnight loops? (no auto-install)
  • Existing group? Stage detects existing feed; clones instead of bootstrap
  • New machine for existing friend? Local wiki starts empty by design (per-friend principle)
04

Identity interview

AI-driven onboarding interview that writes identity.md locally + a public identity summary to the Activity Feed.

  • /sf:interview — about 17 questions across 5 sections
  • About you · Working style · Tech preferences · Opinions / non-goals · Contribution
  • Multiple-choice + open-ended; phase question is informational not mechanical
  • Re-runnable later: /sf:interview --refresh for shifts
  • Writes wiki/identity.md + <activity-feed>/identities/<handle>.md
05

Wiki bootstrap

Create the master wiki skeleton if it doesn't exist yet — directories and page templates only, never content.

  • ~/.claude/sf/wiki/ created with subdirs: research / decisions / alternatives / patterns / projects
  • index.md + log.md seeded
  • Optionally set up a git remote for backup (private repo)
  • /sf:backup pushes wiki to remote on demand
  • Per-friend principle: friend grows their own knowledge from here
06

Doctor verification

Confirm everything works before the friend's first prompt. Optional: turn on Claude Code's native OpenTelemetry for token observability.

  • /sf:doctor reports on every plugin, hook, key, and remote
  • All hooks registered in expected order
  • All API keys present and reachable
  • Optional: OTEL_EXPORTER_OTLP_ENDPOINT + headers for token + session observability
  • Surfaces any drift the next morning if something breaks
07

First-session walkthrough

Quick tour of the commands the friend will actually run, day after day. Then we get out of the way.

  • /sf:wake-up — auto on every session start
  • /sf:wrap — end-of-session consolidate
  • /sf:note <text> · /sf:recall <query>
  • /sf:improve-skill <name> — Karpathy auto-research loop
  • /sf:bootstrap-project · /sf:catch-up · /sf:backup
  • /revise-claude-md — from claude-md-management (CLAUDE.md layer complement to /sf:wrap)

§ 07 · What it stands on

Eight principles
worth being annoying about.

Every architectural choice is downstream of these. When the framework grows past one of them, the principle wins — the feature gets cut.

finis.
Token efficiency is the north star. Every design decision is checked against it.
i. — north star
Per-friend local. Share by action — never by leak.
ii. — privacy default
Curation over kitchen-sink. The framework's value is what it leaves out.
iii. — opinionated
Files and configs only. No daemons at our layer. Plugins may have their own.
iv. — auditable
Friends shouldn't lose work because the framework moved. Schema-versioned migrations.
v. — backwards compatibility
Curation extends to maintainers. Popularity ≠ safety. Vet identity and incentives.
vi. — provider trust
Format constraint as privacy. Trust terseness — not safety nets like diff-review-by-default.
vii. — terse-as-firewall
Small enough to throw away when models evolve past it. Manus rewrote their harness five times in six months.
viii. — disposable
