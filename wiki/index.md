### Foreground web research

- [claude-mem (thedotmack)](research/claude-mem.md) — Cross-session memory plugin for Claude Code (and others). Apache-2.0. Architecture: 5 hooks + Bun worker on port 37777 + SQLite FTS5 + ChromaDB + local ONNX embeddings + 3-layer progressive disclosure retrieval. The most-recommended cross-session memory tool; likely adoption candidate for individual-level memory layer with our wiki layered on top for team synthesis.
- [Context Mode (mksglu)](research/context-mode.md) — Within-session token efficiency plugin via tool-call sandboxing. ELv2 license (restricts SaaS, fine for our use). Per-project SQLite FTS5 + 5 hooks + 11 MCP tools + subprocess-only isolation (no daemon). 315 KB → 5.4 KB (98% reduction). Complementary to claude-mem; together they cover within-session + cross-session efficiency.
- [Superpowers (obra)](research/superpowers.md) — Software development methodology as composable skills, 7-phase workflow (Brainstorm → Worktree → Plan → TDD → Subagent → Review → Finish). MIT license, in Anthropic's official marketplace, 150K+ stars. Currently powering our brainstorming session (dogfooding). Skills include brainstorming, writing-plans, TDD, subagent-driven-development, systematic-debugging, code-review. No hook conflicts with claude-mem / Context Mode.
- [GSD Redux (Get Shit Done)](research/gsd-redux.md) — Spec-driven sub-agent development framework. MIT license, multi-platform. 3-phase workflow + persistent artifacts (PROJECT.md, STATE.md, CONTEXT.md, etc.). Substantial overlap with Superpowers — recommendation: pick Superpowers (already in use), but borrow GSD's persistent-artifact taxonomy for project sub-wikis. Trust note: use open-gsd fork, NOT legacy gsd-build (abandoned over trust concerns).
- [Skill Creator (Anthropic Official)](research/skill-creator.md) — Apache-2.0, 256K+ installs. Four modes (Create / Eval / Improve / Benchmark) + four sub-agents (Executor / Grader / Comparator / Analyzer). Implements Simon Scrapes' "Layer 1" description optimizer (60/40 train/test, 3 runs/query, 5 iterations). Foundation for any skill-shipping framework. Requires ANTHROPIC_API_KEY for description optimizer (caveat).
- [Frontend Design (Anthropic Official)](research/frontend-design.md) — Auto-activating skill for distinctive frontend interfaces. Rejects generic AI aesthetics; teaches bold typography + color palettes + animations. Conditional adoption — include for UI-building projects, skip for backend-only work. Single skill, no hook conflicts. License verification pending.
- [qmd (Tobi Lütke)](research/qmd.md) — Hybrid search engine for local markdown KBs (BM25 + vector + LLM re-rank). MIT, all on-device with ~2GB local GGUF models. **Karpathy's named v2 upgrade path** for the LLM Wiki pattern when index-driven recall stops scaling. NOT for v1; drop-in compatible later if wiki structure stays as plain markdown.
- [Ralph / Ralph Wiggum (Anthropic)](research/ralph.md) — The canonical "lightweight harness" pattern (Geoffrey Huntley: "Ralph is a Bash loop"). Stop-hook-based session loop: re-feeds prompt until completion token output. Documented overnight successes (6 repos in a YC hackathon, $50K contract for $297). Don't bundle; recommend for autonomous task loops. **Surfaces critical design decision**: Stop hook collision with our consolidate skill — resolve by using `/wrap` slash command instead of Stop hook.
- [Memory Architecture Alternatives (Mem0 / Letta / Zep / LangMem)](research/memory-architecture-alternatives.md) — Framework-agnostic memory systems we considered but didn't adopt. All target "add memory to your custom agent framework" — different problem than "auto-memory for Claude Code sessions." claude-mem wins for our use case. Evidence for the memory-architecture ADR's "Alternatives considered" section.
- [**ECC — Everything Claude Code (CRITICAL course-correction)**](research/ecc-everything-claude-code.md) — The plugin we mistakenly dismissed in ADR-006 without inspection. Harness-native operator system, MIT, 140K+ stars, 246 skills + 61 agents + 14 MCPs, three install profiles (minimal/core/full), file-based instincts memory aligned with our wiki philosophy. Substantial overlap with our framework's design. **Pending discussion**: adopt ECC core + layer on top (A), stay the course + reword (B), or hybrid (C). Multiple ADR amendments candidate.
- [Awesome-Claude-Skills Survey (ComposioHQ)](research/awesome-claude-skills-survey.md) — Meta-survey of ~1000+ skills/plugins across 10 categories. Surfaces novel approaches: lean-ctx (Context Mode alt), Mercury/Proton MCP (team-collab gap), Chrome Relay, OpenWeb, recursive-research, great_cto, Septim Agents Pack. Ecosystem scope: 15K+ plugin repos indexed by May 2026. Curation is more valuable, not less.
- [Team-Coordination Plugin Survey (Mercury/Proton vs MCP Agent Mail)](research/team-coordination-survey.md) — Mercury/Proton: polished SaaS, third-party data. **MCP Agent Mail (Dicklesworthstone)**: self-hosted, Git + SQLite + FastMCP, advisory file reservations solve multi-author wiki conflict. Recommendation pending design-doc: adopt MCP Agent Mail as optional onboarding install for friend groups >1 person.
- [lean-ctx (yvgude) — Context Mode Competitor](research/lean-ctx.md) — Apache-2.0+MIT, Rust single binary, 62 MCP tools, tree-sitter AST across 21 languages, knowledge graph w/ temporal validity, browser token dashboard with budget profiles. More comprehensive than Context Mode but newer (4 months, 2.2K stars). 3 positions pending: swap, keep + document, or use both.
- [Anthropic Plugins Marketplace Catalog (Aerial View)](research/anthropic-marketplace-catalog.md) — Survey of ~100+ official plugins. Mostly domain-specific (clouds/databases/CRMs); confirms our framework operates at a different (horizontal infrastructure) layer. Framework-relevant follow-up candidates: dash0 (observability), context7 (version-aware docs), claude-md-management (CLAUDE.md hygiene).
- [Observability Tools (dash0 + native CC + alternatives)](research/observability-tools-survey.md) — **Major reframe**: Claude Code has NATIVE OpenTelemetry support. Token observability story is essentially free — friends set env vars, point at OTLP backend, done. dash0's agent-skills are for friend-group projects that emit OTel (not for observing CC). Native CC + backend = answer.
- [context7 (Upstash) — Version-Specific Documentation Lookup](research/context7.md) — Anthropic-official MCP by Upstash. Pulls up-to-date version-specific docs into Claude Code prompts. Solves model staleness. Auto-activating skill + docs-researcher sub-agent on cheaper model. Strong ADR-006 curated stack amendment candidate. Caveat: requires Upstash API key.

## Decisions (ADRs)

- [ADR-001: Harness vs Framework Terminology](decisions/001-harness-vs-framework-terminology.md) — Internal: "meta-harness configuration." External: "framework" or "agentic OS" stays acceptable. Concrete renaming guide per context.
- [ADR-002: Token-Efficiency Stack](decisions/002-token-efficiency-stack.md) — Adopt Context Mode (within-session) + claude-mem (cross-session) as external plugins; our wiki + future qmd as team-level layer. License burden: ELv2 + Apache-2.0. 5 alternatives rejected with reasoning. *(amended 2026-05-28: lean-ctx documented as Alternative F + sunset triggers)* *(amended C2 2026-06-17: lean-ctx adopted narrowly as code-map CLI engine only — not as memory/token-efficiency replacement; see ADR-035)*
- [ADR-003: No-Daemon Rule (Configuration Layer)](decisions/003-no-daemon-rule.md) — The configuration adds zero daemons of its own (files + Claude Code configs only). Plugins we recommend may have their own (claude-mem's worker, etc.) — different scope. Forbids ours; permits theirs.
- [ADR-004: Wiki Design — Hierarchical Master + Project Sub-Wikis](decisions/004-wiki-design-hierarchical.md) — Master wiki (studio-level shared knowledge) + project sub-wikis under `wiki/projects/<project>/`. CWD-aware loading; target 3-5K tokens of relevant context at session start. Page format conventions established. 4 alternatives rejected.
- [ADR-005: Wiki Retrieval Evolution — index.md Now, qmd at Scale](decisions/005-wiki-retrieval-evolution.md) — v1: index.md-driven, no infra. v2: qmd drop-in when triggers fire (>200 master entries, >500 aggregate project entries, >8K token wake-ups, recurring miss-failures). Pre-conditions for cheap transition enforced now: plain markdown, stable frontmatter, abstracted retrieval logic.
- [ADR-006: Curated Stack — Which Plugins + Which Superpowers Skills](decisions/006-curated-stack.md) — Required: Superpowers + Skill Creator + claude-mem + Context Mode + **context7** + **claude-md-management**. Conditional: Frontend Design. Documented-not-bundled: Ralph + **MCP Agent Mail**. ECC studied for inspiration, not adopted. Superpowers skills split into universal + build-phase-only. *(amended 2026-05-28: context7/claude-md-management added; ECC framing reworded; MCP Agent Mail documented as deferred)*
- [ADR-007: Provider-Vetting Principle (Curate Providers, Not Just Tools)](decisions/007-provider-vetting-principle.md) — Curation must vet BOTH tool fit AND provider trust (identity, governance, history, incentives, maintenance signal). Inspired by GSD's meme-coin rug-pull incident + PY's "1 in 4 community skills contain vulnerabilities." 2-week response timeline if a provider turns adversarial after adoption. 4 alternatives rejected. *(amended 2026-05-28: trust assessment table extended with 5 new providers — ECC, Upstash, Dash0, Mercury, Dicklesworthstone)*
- [ADR-008: Wake-Up Hook — Preserves Prompt Cache, Loads Wiki Into Conversation Layer](decisions/008-wake-up-hook.md) — SessionStart hook injects wake-up context as user-role conversation message, NEVER as system prompt modification (preserves prompt cache). Reads cwd; loads master index + project index/log/session-pointer + master log tail. 3-5K token target. Retrieval abstraction enforced for ADR-005's v2 transition. 5 alternatives rejected. *(amended C2 2026-06-17: code-map is load-on-demand only — the digest is never injected at SessionStart; a single pointer line in the project context is permitted; see ADR-035)*
- [ADR-009: Consolidate via /wrap Slash Command (Not Stop Hook)](decisions/009-consolidate-via-wrap.md) — `/wrap` is user-invoked; no Stop hook (avoids Ralph collision + claude-mem SessionEnd ordering). High-signal-threshold promotion; default = ZERO edits unless real signal. Companion: `/sf:note` + `/sf:recall`. 5 alternatives rejected. *(amended 2026-05-28: noted /revise-claude-md complement at CLAUDE.md layer)*
- [ADR-010: Hook Ordering Coordination — Multi-Plugin Lifecycle Coexistence](decisions/010-hook-ordering.md) — Preferred order: Context Mode → claude-mem → our wake-up (most specific to most general). Our hooks designed for robustness (idempotent, order-insensitive, graceful failure, deterministic). Ralph compatibility documented honestly. 4 alternatives rejected.
- [ADR-011: Skill Schema — Frontmatter + References + Execution Contracts + Evals](decisions/011-skill-schema.md) — Every shipped skill: SKILL.md + references/ + eval/eval.json + optional learnings.md. Binary assertions only in eval.json; subjective review goes to Skill Creator's dashboard. 4 alternatives rejected.
- [ADR-012: Two-Layer Self-Improvement — L1 via Skill Creator + L2 via Karpathy Loop](decisions/012-two-layer-self-improvement.md) — L1 (activation): Skill Creator's description optimizer (adopted). L2 (body quality): new framework skill `/sf:improve-skill <name>` applying Karpathy loop to SKILL.md. Modes: interactive + autonomous (overnight). One change per iteration, binary assertions, git as memory, dedicated branch per run. 4 alternatives rejected.

- [**ADR-034: Cadence as Glue over Native Primitives**](decisions/034-cadence-as-glue.md) — The framework ships **no scheduler/daemon** (ADR-003); cadence = thin glue over native `/loop` / Cron / `/goal` / Cloud Routines. Net-new (framework had zero cadence ADRs); **unblocks C4**. 3 alternatives rejected.
- [**ADR-035: Code-Map Context Layer**](decisions/035-code-map-context-layer.md) — lean-ctx CLI adopted as code-map engine (per-file `read -m signatures`) behind an engine-agnostic `Symbol` contract. Architecture: `lib/codemap/` (models, adapter_leanctx, digest, staleness, sources); regenerable cache under `${CLAUDE_PLUGIN_DATA}/code-maps/`; **never** in the wiki, **never** in the user's repo (ADR-032 read-only discipline). Staleness detection via per-file content hashes + STALE banner. Load-on-demand only — **never** injected at SessionStart (ADR-008 amendment). Graceful-degrade when lean-ctx absent. `/ren:doctor` CODE-MAP check. C5 call-graph extension path preserved in `Symbol` model. CLI-only (`lean-ctx serve` daemon excluded per ADR-003). **Amends ADR-002** (lean-ctx adopted as code-map CLI only, not as memory layer) and **ADR-008** (code-map excluded from wake-up injection). 3 alternatives rejected (universal-ctags, tree-sitter pip, Python ast).
- [**ADR-036: Bike-Method / Earned Autonomy for Self-Modifying Loops**](decisions/036-bike-method-earned-autonomy.md) — Formalizes the earned-autonomy posture for `/ren:improve-skill` now that the eval backend is wired.
