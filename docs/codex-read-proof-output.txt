Falcon is a local-first CLI for tracking reading habits. It is written in Python 3.11 with `uv`, stores data in a single SQLite database at `~/.falcon/db.sqlite`, and has no server component. Source: [map.md](/tmp/renosproof/wiki/projects/falcon/map.md:7).

The three most important decisions are:

1. Use SQLite instead of Postgres because Falcon is single-user and local-first; zero operations matter more than database power. Source: [use-sqlite.md](/tmp/renosproof/wiki/decisions/use-sqlite.md:4).

2. Keep the product local-only with no cloud sync, so user data stays on the machine and Falcon avoids account infrastructure that would conflict with its privacy positioning. Source: [local-only.md](/tmp/renosproof/wiki/decisions/local-only.md:4).

3. Standardize on the Python/`uv` toolchain, with pytest for tests and conventional commits. This is a project convention rather than a formal decision record. Sources: [map.md](/tmp/renosproof/wiki/projects/falcon/map.md:7), [preferences.md](/tmp/renosproof/wiki/global/preferences.md:5).
